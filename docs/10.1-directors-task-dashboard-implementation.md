# Doc 10.1A ‚Äî Directors Task Dashboard (Implementation / ERPNext Setup Guide)

## 1) Purpose
This is a step-by-step guide to implement:
- **Doc 10.1 ‚Äî Directors Task Dashboard (TV / Wallboard)**

It covers:
- Creating a dedicated TV user
- Restricting what the TV can see using **Task Access Policies** (Doc 10 / Doc 10A)
- Building the dashboard using:
  - Option A: saved Task list view wallboard
  - Option B: a Directors Workspace page

---

## 2) Prerequisites / access
Do this as a user with:
- `System Manager`

Also confirm Doc 10A is implemented (Task Access Policy visibility model):
- `Task.task_access_policy` exists
- `Task Access Policy` DocType + records exist
- User Permission filtering is enabled for restricted users (including the TV user)

You will use:
- `User`
- `Role` + `Role Permission Manager`
- `User Permission`
- `Task` list view (filters / grouping)
- `Workspace`

---

## 3) Create a dedicated TV user (recommended)
Why:
- You want a stable login with controlled permissions.
- You do not want a personal director account logged in on a public TV.

Steps:
1) Open `User`.
2) Click `Add User`.
3) Set:
   - Email: `director.tv@company` (example)
   - First Name: `Directors TV`
   - Send Welcome Email: off (recommended if email is not configured)
4) Save.
5) Set a strong password and store it securely.

---

## 4) Permissions for the TV user (least privilege)
The TV user should typically be **read-only**.

### 4.1 Create a role for the TV user
Steps:
1) Open `Role`.
2) Click `New`.
3) Role Name: `Directors TV`.
4) Save.

### 4.2 Assign the role to the TV user
Steps:
1) Open the TV `User` record.
2) In Roles, add:
   - `Directors TV`
3) Save.

### 4.3 Give the role read access to Tasks
Steps:
1) Open `Role Permission Manager`.
2) Select DocType: `Task`.
3) Find role: `Directors TV`.
4) Enable:
   - Read
5) Disable:
   - Write
   - Create
   - Delete
   - Submit
   - Cancel
   - Amend
6) Save.

Notes:
- If you want directors to be able to click into a Task and open attachments, you may need to ensure the TV user can read `File`.
- Keep permissions minimal; add only if something is blocked in practice.

---

## 5) Configure what the TV user can see (Task Access Policies)
Doc 10 requires that Task visibility is controlled by **Task Access Policies**.

### 5.1 Decide which Task Access Policies appear on the TV
Common choices:
- Show all operational execution tasks (Delivery/Inventory/Accounting queues)
- Hide director-only tasks (Debt Collection / Approvals) if the TV is in a public area

This decision is implemented by granting (or not granting) the relevant Task Access Policies to the TV user.

### 5.2 Grant the TV user access to Task Access Policies
Steps:
1) Open `User Permission`.
2) Click `New`.
3) Set:
   - User: the TV user
   - Allow: `Task Access Policy`
   - For Value: pick one policy (example: `Delivery`)
4) Save.
5) Repeat for each policy you want the TV user to see.

### 5.3 Verify user-permission filtering is enabled on Task
If user-permission filtering is not enabled, the TV may see tasks it should not.

Steps:
1) Open `Role Permission Manager`.
2) Select DocType: `Task`.
3) For each role that must be restricted (including `Directors TV`), ensure:
   - `Apply User Permissions` is enabled
4) Save.

## 6) Option A ‚Äî Saved Task list view wallboard (step-by-step)
This option is the fastest path to a working TV wallboard.

### 6.1 Create the ‚ÄúAll Open Tasks‚Äù wallboard view
Steps:
1) Open the `Task` list.
2) Click `Filters`.
3) Add filters:
   - `Status` not in `Completed, Cancelled`
4) Set visible columns:
   - Assigned To
   - Task Kind
   - Subject
   - Status
   - Modified
5) Set grouping:
   - Click `Group By` ‚Üí select `Assigned To`
6) Save the view:
   - Click `Save` (or `Save as`)
   - Name: `Directors ‚Äî Wallboard (All Open)`
   - Save as a shared view if you want it visible to multiple director users

### 6.2 Create per-team saved views (recommended)
These make it easy for directors to drill down.

Create a saved view per team by adding a filter:
- `Task Kind` equals the relevant kind(s)

Suggested saved views:
- `Delivery ‚Äî Open Tasks` (Delivery + Pickup Returns + Return drop-off)
- `Inventory ‚Äî Open Tasks` (Pack / prepare + Returns processing)
- `Accounting ‚Äî Open Tasks` (Invoice preparation)

Also create director escalation views:
- `Directors ‚Äî Open Tasks` (Debt Collection + approvals + Distribute Payment)

---

### 6.3 Recommended saved filters (templates from Doc 10.1)
Create these saved Task filters. These power both Option A (wallboard) and Option B (Workspace).

#### 6.3.1 Global
Create a saved view:
1) Open `Task` list.
2) Filters:
   - `Status` not in `Completed, Cancelled`
3) Save view name: `Directors ‚Äî All Open Tasks`

#### 6.3.2 By team
Create these saved views (each should also include the Status filter above):

1) `Order Accepting ‚Äî Open Tasks`
   - Filter: `Task Kind` = `Order entry`

2) `Inventory ‚Äî Open Tasks`
   - Filter: `Task Kind` in:
     - `Pack / prepare items`
     - `Returns processing / verification`

3) `Delivery ‚Äî Open Tasks`
   - Filter: `Task Kind` in:
     - `Dispatch picking / hand-off`
     - `Delivery`
     - `Return to warehouse (aborted delivery / cancelled order)`
     - `Pickup Returns`
     - `Return drop-off at warehouse`

4) `Accounting ‚Äî Open Tasks`
   - Filter: `Task Kind` = `Invoice preparation / create invoice`

5) `Directors ‚Äî Open Tasks`
   - Filter: `Task Kind` in:
     - `Debt Collection`
     - `Distribute Payment`
     - `Discount Approval`
     - `Purchase Approval`
     - `Write-off Approval`

#### 6.3.3 By urgency
These views are optional, but if you create them, use these exact definitions.

1) `Overdue Tasks`
   - Only if you use due dates:
     - Filter: `Expected End Date` < Today
     - Filter: `Status` not in `Completed, Cancelled`

2) `Stuck Tasks`
   - Default go-live definition: tasks that are Open/Working and not modified for 8 hours.
   - Filters:
     - `Status` in `Open, Working`
     - `Modified` < Now - 8 hours
   - Save view name: `Stuck Tasks`

## 7) Option B ‚Äî Directors Workspace (step-by-step)
A Workspace is a ‚Äúhome page‚Äù with shortcuts.

### 7.1 Create a Workspace
Steps:
1) Open `Workspace`.
2) Click `New`.
3) Title: `Directors ‚Äî Operations`.
4) Save.

### 7.2 Add shortcuts to saved Task views
Steps:
1) Open your new Workspace.
2) Click `Edit`.
3) Add shortcuts/links to (use the saved views from section 6.3):
   - `Directors ‚Äî All Open Tasks`
   - `Delivery ‚Äî Open Tasks`
   - `Inventory ‚Äî Open Tasks`
   - `Accounting ‚Äî Open Tasks`
   - `Directors ‚Äî Open Tasks`
   - `Stuck Tasks` (optional)
4) Save.

Note:
- Keep it simple. The primary value is one-click access to each queue.

---

## 8) TV / kiosk setup (practical)
These steps are not ERPNext-specific, but they are required to keep the TV stable.

Steps:
1) On the TV computer, open the ERPNext URL and log in as the TV user.
2) Open the saved wallboard view (Option A) or the Directors Workspace (Option B).
3) Enable full-screen mode in the browser.
4) Ensure the PC does not sleep.

Optional:
- If the page does not auto-refresh, use a kiosk approach that refreshes periodically.

---

## 9) Validation checklist
Access control:
- TV user can open Task list.
- TV user cannot edit Tasks.
- TV user only sees tasks allowed by its Task Access Policy permissions.

Wallboard usability:
- Group-by `Assigned To` works.
- Saved views load correctly.
- Directors can drill down into team queues.
