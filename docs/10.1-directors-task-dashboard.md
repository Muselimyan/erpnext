# Doc 10.1 — Directors Task Dashboard (TV / Wallboard)

## 1) Purpose
This document defines how directors can view operational execution in real time (or near real time) using ERPNext Tasks.

Goals:
- Show a clear list of what work is open and who owns it.
- Be usable on a TV (kiosk) in the directors’ office.
- Respect your Task Access Policy model (Doc 10 / Doc 10A): the TV user only sees what you allow.

Non-goals:
- This doc does not define business workflows.
- This doc does not require custom development (but notes optional enhancements).

---

## 2) Inputs / assumptions
- You use **Tasks for every operational stage** (Doc 10).
- Visibility is controlled by **Task Access Policies** (Doc 10 section 6.2 + Doc 10A).
- Completion/edit rights are controlled by owning-team enforcement (Doc 10 section 6.1).

---

## 3) What directors want to see (minimum)
Wallboard should answer:
- Which tasks are open right now?
- Who is responsible?
- What is late / stuck?

Minimum columns:
- Assigned To
- Task Kind
- Subject
- Status
- Modified (or Due Date if you use it)

Recommended filters:
- Exclude `Completed` and `Cancelled` tasks
- Optionally exclude low-value task kinds (if you add any later)

---

## 4) Access & security rules (TV user)
Create a dedicated user for the TV:
- Example user: `director.tv@company`

Access rules:
- Grant only the minimum permissions to view Tasks and dashboards.
- Control which tasks appear using **Task Access Policy** permissions:
  - If the TV should show everything operational, grant the TV user access to all Task Access Policies.
  - If the TV should hide director-only tasks, do not grant those policies.

Practical note:
- If you use User Permission filtering on Task (Doc 10A), the TV view is automatically “safe” as long as you only grant the correct policies.

---

## 5) Option A — Saved Task list view wallboard (recommended starting point)
This is the simplest wallboard: a filtered `Task` list view with saved filters.

### 5.1 Setup
1) Open `Task` list.
2) Apply filters:
   - `Status` not in (`Completed`, `Cancelled`)
   - (Optional) `Task Kind` in (your operational task kinds)
3) Choose columns:
   - Assigned To
   - Task Kind
   - Subject
   - Status
   - Modified (or Due Date)
4) Grouping:
   - Group By: `Assigned To`
5) Save as a named filter:
   - Example: `Directors — Wallboard (All Open)`

### 5.2 TV display
- Open the saved filter on the TV.
- Use full-screen/kiosk mode.

### 5.3 Strengths
- Fast to set up
- No customization
- Easy to adjust filters/columns

### 5.4 Limitations
- Not a perfect “dashboard”; it is a list.
- Auto-refresh behavior depends on the browser/ERPNext behavior.
  - If the page does not refresh on its own, use a safe kiosk refresh approach.

---

## 6) Option B — Directors Workspace dashboard page
This is a “director home page” that links to saved task views and shows quick counts.

### 6.1 Setup
Create a Workspace (example: `Directors — Operations`).

Include:
- A link card to:
  - `Directors — Wallboard (All Open)`
- Link cards per team (each is a saved Task filter):
  - Order accepting — open
  - Inventory — open
  - Delivery — open
  - Accounting — open
  - Directors (approvals/debt) — open
- Optional count-type shortcuts (if you use them):
  - Overdue tasks
  - Tasks stuck in Open for more than N hours

### 6.2 TV usage
- Keep the TV on the Workspace page.
- Directors can click into any queue for details.

### 6.3 Strengths
- More “executive-friendly” than a raw task list
- Works well as a navigation hub

### 6.4 Limitations
- Still not a true live wallboard without refresh
- Requires designing multiple saved filters/views

---

## 7) Recommended saved filters (templates)
Create these saved Task filters to power both options:

Global:
- `Directors — All Open Tasks`

By team:
- `Order Accepting — Open Tasks`
- `Inventory — Open Tasks`
- `Delivery — Open Tasks`
- `Accounting — Open Tasks`
- `Directors — Open Tasks`

By urgency:
- `Overdue Tasks` (if due dates are used)
- `Stuck Tasks` (example definition: Open/Working for more than N hours)

---

## 8) Optional enhancements (later)
If you later want a more “TV-grade” experience:
- A custom page that auto-refreshes every N seconds
- A custom report that shows:
  - worker → count of open tasks
  - oldest task age
  - overdue count
  - clickable drill-down

These are not required for go-live.

---

## 9) Acceptance criteria
- Directors can open one page and see all open work grouped by assignee.
- Directors can quickly drill down into team queues.
- The TV user sees only allowed tasks (via Task Access Policies).
